\begin{abstract}
This document records the current architecture and mathematical contract of the
\stl{} repository after consolidation. The live project is a compact Python
implementation of the Drop the Handkerchief game organized around engine
truth, exact/search solver anchors, value/policy learning, gated self-play, and
playable agents. The solver models the game as a finite stochastic zero-sum
public-state game with simultaneous exact-second actions at each half-round.
The rigorous core is finite-horizon minimax over engine-derived transitions.
The approximate core adds selective candidate sets, matrix-game MCTS, Tier A
interval tablebase lookup, learned leaf evaluation, calibration gates, and a
bridge toward AlphaZero-style self-play. The current code layout is deliberately
small: \path{stl/solver} contains only \path{exact.py}, \path{search.py}, and
\path{tablebase.py}.
\end{abstract}

\section{Canonical Game Model}

\subsection{Public State}

Let the players be \(i \in \{\hal,\baku\}\). A public state
\(s \in \states\) is represented by player cylinders, total time dead, death
counts, life status, referee CPR count, game clock, current half-round, round
index, first dropper, terminal status, and winner. In code, this is the
\path{ExactPublicState} in \path{stl/solver/exact.py}.

The opening state uses game-clock \(t=720\), corresponding to 8:12:00 AM. The
leap-second window is the inclusive interval \([3540,3600]\), corresponding to
8:59:00 through the inserted 8:59:60 second.

\subsection{Actions}

Each half-round has a dropper \(D(s)\) and checker \(C(s)\). The simultaneous
action is
\[
  a=(d,c), \qquad d \in A_D(s), \quad c \in A_C(s),
\]
where \(d\) is the drop second and \(c\) is the check second.

Normal turns have duration 60. Leap-window turns have duration 61. The
actor-aware legality rule is:
\[
\begin{aligned}
  A_D(s) &= \{1,\ldots,60\} && \text{if } D(s)=\hal,\\
  A_D(s) &= \{1,\ldots,T(s)\} && \text{if } D(s)=\baku,\\
  A_C(s) &= \{1,\ldots,60\} && \text{for both players,}
\end{aligned}
\]
where \(T(s)=61\) only in the leap window and \(T(s)=60\) otherwise. Baku's
only second-61 privilege is dropping during the leap window. Hal can never
drop or check on second 61.

\subsection{Resolution and Chance}

For action \(a=(d,c)\), the checker succeeds iff \(c \ge d\). On success,
squandered time is
\[
  \Delta x_C=c-d.
\]
The diagonal case \(c=d\) is a successful check with \(\Delta x_C=0\).
On failure, the checker receives the fixed failed-check penalty
\[
  \Delta x_C=60.
\]
If the checker cylinder reaches or exceeds 300 seconds, or if the checker
failed, the checker enters a death episode with duration
\[
  z=\min(x_C+\Delta x_C,300).
\]

The engine referee computes survival probability as
\[
p_{\mathrm{survive}}(i,z,s)
 =
\left[1-\left(\frac{z}{300}\right)^3\right]_+
\cdot 0.85^{\tau_i/60}
\cdot \max(0.4,0.88^k)
\cdot \rho_i,
\]
with \(\rho_\hal=1.0\) and \(\rho_\baku=0.94\). The probability is zero at
\(z \ge 300\). The solver does not reimplement chance; it derives chance
branches from the engine.

\subsection{Utility}

The rigorous solver uses terminal-only utility from Hal's perspective:
\[
u_\hal(s)=
\begin{cases}
  +1, & \text{Hal is the terminal winner},\\
  -1, & \text{Baku is the terminal winner},\\
  0, & \text{draw or neutral terminal state}.
\end{cases}
\]
Nonterminal frontier states beyond a finite horizon are recorded as unresolved
mass. They are not reward-shaped in the exact solver.

\section{Exact Finite-Horizon Solver}

\begin{assumption}[Engine authority]
The Python engine in \path{stl/engine} is the rule oracle. Solver correctness
is relative to that engine's transition and legality functions.
\end{assumption}

\begin{definition}[Finite-horizon value]
Let \(\valuefn_h(s)\) be Hal's value at state \(s\) with \(h\) half-rounds
remaining in exact recursion. If \(s\) is terminal, then
\(\valuefn_h(s)=u_\hal(s)\). If \(h=0\) and \(s\) is nonterminal, then the
returned scalar value is 0 and the unresolved probability mass is 1.
\end{definition}

For nonterminal \(s\) and \(h>0\), the exact solver forms a Hal-payoff matrix
\(M_h(s)\):
\[
M_h(s)_{d,c}
 =
\expect_{\omega \sim P(\cdot \mid s,d,c)}
\left[
  \valuefn_{h-1}(T(s,d,c,\omega))
\right].
\]
The matrix entries are computed by stepping the authoritative engine and
weighting survival/fatal branches by referee probabilities.

At each simultaneous move the solver uses the standard zero-sum matrix-game LP:
\[
\begin{aligned}
\max_{\sigma,v}\quad & v \\
\mathrm{s.t.}\quad & \sum_i \sigma_i M_{ij} \ge v
  \quad \forall j,\\
& \sum_i \sigma_i = 1,\\
& \sigma_i \ge 0.
\end{aligned}
\]

The LP path is the certified exact path used for finite-horizon labels, audits,
and tablebase verification. A separate CFR+ matrix solver is available for
bounded critical-state resolves, where the goal is fast local re-solving rather
than a new certificate.

\begin{proposition}[Exact finite-horizon value]
For a fixed engine state \(s\), fixed horizon \(h\), and full legal exact-second
action sets, \path{solve_exact_finite_horizon} returns the value of the
finite-horizon stochastic zero-sum game induced by the engine and terminal
utility.
\end{proposition}

\begin{proof}
Terminal and horizon-zero cases are direct. For \(h>0\), every legal joint
action is expanded through the engine. Death branches are weighted by referee
probabilities. The resulting one-step object is a finite zero-sum matrix game,
whose LP value equals its minimax value by von Neumann's minimax theorem.
Induction on \(h\) gives the finite-horizon stochastic-game value.
\end{proof}

\section{Search and Approximation}

\subsection{Selective Exact Search}

Full-width exact search is expensive: normal turns have \(60 \times 60\) joint
actions and leap-window Baku-dropper turns have \(61 \times 60\). The selective
solver in \path{stl/solver/search.py} keeps critical seconds, local diagonals,
safe-check boundaries, and overflow thresholds. It exposes audit helpers for
comparing selective values against full-width values on states where full-width
search is tractable.

The tight selective candidate set is used by exact-label and gate workflows.
Live human play uses a broader root grid so that the playable agent is not
pattern-readable around only endpoint seconds.

\subsection{Matrix-Game MCTS}

The MCTS layer is a simultaneous-move tree. Each node stores candidate drop
seconds, candidate check seconds, a \(Q\)-matrix, visit counts, priors, a game
snapshot, and child edges. Selection solves a matrix game over an
exploration-adjusted \(Q\)-matrix and samples a joint action. Expansion samples
engine chance branches. Backup keeps all values in Hal perspective, so no sign
flip is needed.

Leaf evaluators can be:

\begin{itemize}[leftmargin=*]
  \item terminal-only;
  \item pinned tablebase plus fallback;
  \item value/policy network;
  \item Tier A interval lookup plus fallback.
\end{itemize}

\subsection{Bounded Critical Resolve}

Critical states near overflow, leap-window timing, or LSR routing can trigger a
fresh root resolve. The default resolve horizon is one half-round: the current
simultaneous move is solved over exact-second candidates and the frontier is
anchored by the leaf evaluator. This is the DeepStack-style depth limit used to
avoid unbounded terminal search. The local matrix can be solved with CFR+ while
exact-label and audit workflows continue to use the LP solver.

The resolve does not introduce a hidden leap-second nature node or a separate
survival-shaped terminal utility. Leap legality remains actor-aware engine/play
truth, and terminal utility remains Hal-perspective win/loss.

\subsection{Tier A Tablebase}

Tier A is an interval-valued post-leap tablebase over states with at most one
total death. It is not a replacement for the exact solver. It is a certified
frontier/lookup layer used to anchor search and best-response probes where
available. Its build and runtime lookup code live in \path{stl/solver/tablebase.py}.

\section{Learning and Self-Play Bridge}

\path{stl/learning} is the exact-to-self-play bridge:

\begin{itemize}[leftmargin=*]
  \item \path{targets.py}: feature extraction, exact label generation, corpus IO;
  \item \path{model.py}: value/policy network and checkpoint-compatible model;
  \item \path{train.py}: supervised training and checkpoint loading;
  \item \path{gates.py}: promotion-facing calibration, audit, ladder, and strength gates;
  \item \path{support/}: bootstrap, reanalysis, self-play, tournaments, route math, and strength internals.
\end{itemize}

The current network consumes a 23-dimensional feature vector and emits one
scalar value plus 124 policy logits: length-62 dropper and checker vectors.
Index 0 is always-illegal padding; index \(s\) is literal second \(s\).
Illegal actions are masked before policy use.

The intended AlphaZero-style loop is:

\begin{enumerate}[leftmargin=*]
  \item generate exact and tablebase anchors;
  \item train a value/policy network;
  \item run search using the network at leaves;
  \item record root values and root policies as bootstrap/self-play targets;
  \item merge exact anchors back into the corpus;
  \item train the next candidate;
  \item promote only if calibration, ladder, strength, and exploitability checks pass.
\end{enumerate}

This is an expert-iteration loop rather than a claim that the full starting
position is solved. Exact finite-horizon outputs are solves. The learned agent
is a calibrated search policy unless and until global exploitability is bounded.

\section{Playable Agent}

The current game CLI loads \path{stl.play.agent.SolverAgent} by default. The
agent combines:

\begin{itemize}[leftmargin=*]
  \item trained value/policy checkpoint;
  \item matrix-game MCTS;
  \item optional Tier A lookup;
  \item stochastic action sampling from the root mixed strategy.
\end{itemize}

For human-facing play, the root candidate grid includes interior seconds so the
agent is not trivially readable as an endpoint-only policy. The CLI also uses a
fresh random seed by default; passing \path{--seed N} restores reproducibility.

Example:
\[
\texttt{uv run python -m stl.cli command=play 'command.args=["--use-tier-a"]'}
\]

\section{What Is Rigorous and What Is Approximate}

Rigorous components:

\begin{itemize}[leftmargin=*]
  \item engine-derived transition model, assuming the engine is the rule oracle;
  \item exact finite-horizon full-width solver;
  \item LP minimax at simultaneous exact-second nodes;
  \item terminal-only utility in the exact core;
  \item pinned tactical scenarios whose expected values are exact by construction.
\end{itemize}

Approximate components:

\begin{itemize}[leftmargin=*]
  \item finite-horizon cutoff and unresolved mass;
  \item selective candidate generation;
  \item MCTS sampling variance and finite iteration budgets;
  \item learned value/policy approximation error;
  \item self-play/bootstrap labels;
  \item empirical calibration and promotion gates.
\end{itemize}

The most important scientific gap is exploitability. Low held-out MSE means the
network predicts the current target corpus well; it does not by itself prove a
low-exploitability strategy over the full game tree.

\section{Validation Status}

After the current consolidation, the verified regression suite collected 629
tests and passed in full. These tests cover engine mechanics, leap-second
legality, exact solver behavior, selective search, MCTS, tablebase fixtures,
Tier A lookup, training glue, calibration, strength gates, play adapters, and
the CLI-facing agent paths.

The regression suite is necessary but not sufficient for a solver paper. The
next evidence needed is:

\begin{itemize}[leftmargin=*]
  \item broader held-out tactical anchors;
  \item candidate-regret audits on larger state samples;
  \item MCTS seed variance and confidence intervals;
  \item best-response or exploitability estimates for promoted checkpoints;
  \item policy-head calibration, not only value-head MSE;
  \item principal-line validation against canonical scenarios.
\end{itemize}

\section{Conclusion}

The repository now has a compact and coherent skeleton for a rigorous
search-and-learning agent: engine authority, exact finite-horizon minimax,
selective search, matrix-game MCTS, Tier A tablebase anchors, a value/policy
network, gated promotion, and a playable solver-backed CLI. The code is not
claiming the game is solved globally. It is a calibrated exact/search/learning
system with a clear path toward stronger AlphaZero-style self-play and
exploitability evidence.
